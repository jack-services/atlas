# Feature Issue Template

Use this template when Atlas creates feature implementation issues.

## Template

```markdown
## Context

{Why this feature is needed and how it aligns with company goals}

## Overview

{Brief description of the feature and its expected behavior}

## Tasks

- [ ] {Task 1}
- [ ] {Task 2}
- [ ] {Task 3}
- [ ] Write/update tests
- [ ] Update documentation if needed

## Acceptance Criteria

- [ ] {Criterion 1 - specific, testable}
- [ ] {Criterion 2}
- [ ] {Criterion 3}
- [ ] All tests pass
- [ ] No regressions introduced

## Verification Steps

1. {How to verify criterion 1}
2. {How to verify criterion 2}
3. Run the test suite: `{test command}`

## Technical Notes

### Relevant Files

- `{path/to/file1}` - {why relevant}
- `{path/to/file2}` - {why relevant}

### Related Knowledge

{Snippets from knowledge base if relevant}

### Dependencies

- {Any dependencies on other issues}
- {External dependencies}

## Labels

- `type:feature`
- `{area label based on files affected}`

---
*Created by Atlas /plan command*
```

## Usage Guidelines

1. **Context**: Connect the feature to business value or user needs
2. **Tasks**: Break down into atomic, completable units
3. **Acceptance Criteria**: Must be specific and testable
4. **Verification**: Provide concrete steps to verify completion
5. **File References**: Include line numbers when helpful
